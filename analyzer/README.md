# Анализатор аудио (ZIO + Swing)

Программа читает WAV-файлы, применяет быстрое преобразование Фурье (FFT)
для анализа частотного спектра, детектирует биты, считает темп (BPM),
анализирует распределение энергии по частотным диапазонам и показывает
результат в графическом окне, а также сохраняет JSON-отчёт и текстовую
сводку.

Эффекты построены на **ZIO 2**; собственные наивные монады из блока 0
сохранены как обязательная учебная демонстрация.

Работает в паре с проектом `visualizer`, который читает JSON-отчёт.

## Запуск

```bash
# Графическое окно (интерфейс "view") — рекомендуемый способ
sbt "run --ui"

# Окно + сразу открыть файл
sbt "run --ui track.wav"

# Консоль: анализ с настройками по умолчанию, сохраняет report.json/report.txt
sbt "run track.wav"

# Консоль с параметрами
sbt "run track.wav --fft 2048 --threshold 1.5 --json out.json"

# Интерактивный режим — программа сама спросит путь
sbt run
```

### Опции

| Опция | По умолчанию | Назначение |
|---|---|---|
| `--fft <число>` | 1024 | размер окна FFT (степень двойки) |
| `--threshold <число>` | 1.3 | порог детекции битов (энергия / средняя) |
| `--window <число>` | 43 | окно усреднения энергии в кадрах |
| `--json <путь>` | report.json | путь к JSON-отчёту (только в консольном режиме) |
| `--text <путь>` | report.txt | путь к текстовому отчёту (только в консольном режиме) |
| `--ui` | — | открыть графический интерфейс |

### Работа в графическом окне

1. Нажать **«Открыть WAV…»** — выбрать WAV-файл, анализ запустится автоматически.
2. Дождаться результата — в окне отобразятся графики спектра, таймлайн битов, BPM.
3. Нажать **«Сохранить отчёт…»** — выбрать путь для `report.json`; рядом автоматически сохранится `.txt`. JSON используется визуализатором.

## Возможности

- Парсинг WAV (PCM 16 бит, моно/стерео, формат Extensible)
- БПФ с окном Хэннинга
- Адаптивная детекция битов по скользящему среднему энергии
- Расчёт BPM по медиане интервалов + нормализация в музыкальный диапазон (70–180)
- Анализ энергии по диапазонам (бас 30–250 Hz, средние 250–4000 Hz, высокие 4000+ Hz)
- Графическое окно: огибающие энергии, таймлайн битов, сводка
- Сохранение отчёта из окна через кнопку «Сохранить отчёт…»
- Жанровая характеристика по спектральному балансу

## Архитектура: ZIO + собственные монады

Проект совмещает две вещи, как требует ТЗ:

1. **Блок 0 — собственные наивные монады** (`monads/`): `Monad`, `Reader`,
   `Writer`, `State`, `IO`. Это учебные реализации, показывающие, что мы
   понимаем структуру монад.
2. **Предметная часть и сценарий — ZIO 2**: промышленный эффект-тип.

### Где что используется

| Блок ТЗ | Наивная монада | Чем заменено / где осталось |
|---|---|---|
| Блок 1 (Reader) | `Reader[Config, A]` | заменён на ZIO-окружение `ZIO[Config, E, A]` + `ZLayer` (`Config.layer`). См. `BandAnalyzer`, `Report` |
| Блок 2 (Writer) | `Writer[Vector[String], A]` | **оставлен буквально** в `BeatDetector.detectAll` и `BandAnalyzer.summary`; лог разворачивается в ZIO-консоль через `AnalysisPipeline.emitLog` |
| Блок 3 (State) | `State[S, A]` | **оставлен** в `BeatDetector` (состояние детектора); в UI его роль играет `zio.Ref` (`AnalyzerWindow`) |
| Блок 4 (IO) | наивный `IO[A]` | заменён на `ZIO` / `ZIOAppDefault`; наивный `IO` остаётся в `monads/IO.scala` как демонстрация блока 0 |

### Почему так

- Наивный `Reader` — это `Env => A`. ZIO-окружение `ZIO[R, E, A]` —
  то же самое, но с типизированными ошибками и управлением ресурсами.
  `ZIO.service[Config]` = `Reader.ask`, `ZLayer.succeed(cfg)` = подстановка
  окружения.
- Наивный `IO` без trampoline переполняет стек на длинных цепочках
  `flatMap` (это отмечено и в исходной версии проекта). ZIO имеет
  trampoline и безопасен; поэтому реальный сценарий — на ZIO.
- `zio.Ref` — потокобезопасная изменяемая ячейка; в UI она заменяет
  глобальную переменную «идёт ли анализ», что соответствует духу блока 3.
- `Writer` оставлен в двух местах намеренно — ТЗ просит «в паре мест
  пусть будет». Это чистые вычисления, накапливающие диагностический лог.
- BPM нормализуется в диапазон 70–180: детектор по энергии подвержен
  octave error — он может срабатывать и на сильных долях, и на слабых
  между ними, давая темп вдвое выше реального. Пока BPM выше 180 —
  делим пополам; пока ниже 70 — удваиваем. В лог пишутся обе цифры
  (сырая и нормализованная), чтобы можно было проверить.

## Структура проекта

```
src/main/scala/
├── monads/              # Блок 0: собственные наивные монады
│   ├── Monad.scala
│   ├── Reader.scala
│   ├── Writer.scala
│   ├── State.scala
│   └── IO.scala
├── domain/              # Предметная область (на ZIO)
│   ├── Audio.scala            # типы данных, Config + ZLayer
│   ├── FFT.scala              # БПФ (чистые вычисления)
│   ├── WavReader.scala        # парсер WAV (ZIO Task)
│   ├── BeatDetector.scala     # детекция битов (свои State + Writer)
│   ├── BandAnalyzer.scala     # анализ диапазонов (ZIO-окружение + Writer)
│   ├── Report.scala           # JSON/текст (ZIO-окружение), сохранение (ZIO Task)
│   └── AnalysisPipeline.scala # сборка конвейера на ZIO
├── ui/                  # Графический интерфейс ("view")
│   ├── SpectrumPanel.scala    # отрисовка результата (Swing)
│   └── AnalyzerWindow.scala   # окно, связка Swing ↔ ZIO
└── app/
    └── Main.scala       # ZIOAppDefault: CLI + флаг --ui
```
