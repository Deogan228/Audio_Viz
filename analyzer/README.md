# Анализатор аудио

Курсовая по функциональному программированию (Scala 3).

Программа читает WAV-файлы, применяет быстрое преобразование Фурье (FFT)
для анализа частотного спектра, детектирует биты, считает темп (BPM),
анализирует распределение энергии по частотным диапазонам
и сохраняет результат в виде JSON-отчёта и текстовой сводки.

Работает в паре с проектом `visualizer`, который читает JSON-отчёт
и визуализирует спектр со звуком в реальном времени.

## Запуск

```bash
# Простой случай: путь к файлу и настройки по умолчанию
sbt "run track.wav"

# С параметрами:
sbt "run track.wav --fft 2048 --threshold 1.5 --json out.json"

# Интерактивный режим — программа сама спросит путь:
sbt run
```

### Опции

| Опция | Значение по умолчанию | Назначение |
|---|---|---|
| `--fft <число>` | 1024 | размер окна FFT (степень двойки) |
| `--threshold <число>` | 1.3 | порог детекции битов (энергия / средняя) |
| `--window <число>` | 43 | окно усреднения энергии в кадрах |
| `--json <путь>` | report.json | путь к JSON-отчёту |
| `--text <путь>` | report.txt | путь к текстовому отчёту |

## Возможности

- Парсинг WAV (PCM 16 бит, моно/стерео, формат Extensible)
- БПФ с окном Хэннинга
- Адаптивная детекция битов по скользящему среднему энергии
- Расчёт BPM по медиане интервалов между битами
- Анализ энергии по диапазонам (бас 30–250 Hz, средние 250–4000 Hz, высокие 4000+ Hz)
- Анализ доминирующей частоты по времени
- Сохранение результатов в JSON и в текстовый отчёт
- Жанровая характеристика по спектральному балансу

## Архитектура

Проект построен на собственных монадах без внешних библиотек:

| Монада | Назначение |
|---|---|
| `Monad[M[_]]` | базовая абстракция, операции `pure`, `flatMap`, `map` |
| `Reader[Env, A]` | проброс конфигурации без явной передачи параметров |
| `Writer[Log, A]` | накопление логов анализа |
| `State[S, A]` | состояние детектора битов (история энергий, время последнего бита) |
| `IO[A]` | взаимодействие с файлами и консолью |

### Где какая монада применяется

- **Reader** — параметры FFT, границы диапазонов и параметры детектора битов
  пробрасываются через `Reader[Config, ...]` в `FFT.analyze`, `BandAnalyzer`,
  `Report.buildJson`, `Report.buildText`
- **Writer** — логи накапливаются в `WavReader.read`, `BeatDetector.detectAll`,
  `BandAnalyzer.summary`
- **State** — `BeatDetector` использует `State[DetectorState, ...]` для
  скользящего окна энергий
- **IO** — вход/выход в `WavReader.read`, `Report.save`, весь `Main.scala`

## Структура проекта

```
src/main/scala/
├── monads/              # Блок 0: реализация монад
│   ├── Monad.scala
│   ├── Reader.scala
│   ├── Writer.scala
│   ├── State.scala
│   └── IO.scala
├── domain/              # Предметная область
│   ├── Audio.scala          # типы данных, Config
│   ├── WavReader.scala      # парсер WAV
│   ├── FFT.scala            # БПФ
│   ├── BeatDetector.scala   # детекция битов, BPM
│   ├── BandAnalyzer.scala   # анализ частотных диапазонов
│   └── Report.scala         # формирование JSON и текстовых отчётов
└── app/
    └── Main.scala       # CLI, IO-сценарий
```
