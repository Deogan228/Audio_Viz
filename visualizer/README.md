# Визуализатор аудио

Курсовая работа по Функциональному программированию. 
Работа сделана на языке программирования Scala с использованием ZIO + Swing.

### Главная функция программы

Программа читает JSON-отчёт, созданный анализатором аудио, проигрывает
исходный WAV-файл и синхронно с воспроизведением рисует спектр в
графическом окне. Поддерживает несколько режимов визуализации и
индикатор бита.

Эффекты построены на **ZIO 2**.
Собственные наивные монады сохранены как учебная демонстрация.

Работает в паре с проектом `analyzer`.
Формат отчёта описан в `REPORT_FORMAT.md`.

## Запуск

```bash
# Окно с выбором JSON-отчёта и WAV-файла
sbt "run --ui"

# Сразу указать пути — откроется окно визуализации
sbt "run ../analyzer/report.json ../analyzer/track.wav"

# Другой режим визуализации
sbt "run report.json track.wav --mode bars"

# Без аргументов — откроется окно выбора файлов
sbt run
```

### Опции

| Опция | По умолчанию | Назначение |
|---|---|---|
| `--mode <spectrum\|bars\|spectrogram>` | spectrum | режим визуализации |
| `--width <число>` | 820 | ширина окна |
| `--height <число>` | 460 | высота окна |
| `--fps <число>` | 30 | частота кадров |
| `--ui` | — | принудительно открыть окно выбора файлов |

### Режимы

- **spectrum** — три зоны энергии (бас/средние/высокие) во всю ширину
- **bars** — три отдельных столбика с зазорами
- **spectrogram** — пока рисуется как spectrum (задел на будущее)

## Возможности

- Чтение JSON-отчёта от анализатора (собственный парсер JSON без библиотек)
- Воспроизведение WAV через `javax.sound.sampled`
- Синхронизация визуализации с реальной позицией воспроизведения
- Графическое Swing-окно с анимацией ~30 кадров/с
- Индикатор бита (вспышка `BEAT`)
- Отображение времени, BPM, режима

## Архитектура: ZIO + собственные монады

Проект совмещает две вещи, как требует ТЗ:

1. **Блок 0 — собственные наивные монады** (`monads/`): `Monad`, `Reader`,
   `Writer`, `State`, `IO`. Учебные реализации.
2. **Предметная часть и сценарий — ZIO 2**.

### Где что используется

| Блок ТЗ | Наивная монада | Чем заменено / где осталось |
|---|---|---|
| Блок 1 (Reader) | `Reader[RenderConfig, A]` | заменён на ZIO-окружение `ZIO[RenderConfig, E, A]` + `ZLayer` (`RenderConfig.layer`). См. `Renderer`, `Animation` |
| Блок 2 (Writer) | `Writer[Vector[String], A]` | **оставлен буквально** в `ReportReader.read`; лог разворачивается в консоль в `VisualizerWindow` / `Main` |
| Блок 3 (State) | `State[S, A]` | состояние анимации `AnimationState` живёт в `zio.Ref`; обновляется атомарно в `Renderer.tickAnimation`. Наивный `State` остаётся в `monads/` |
| Блок 4 (IO) | наивный `IO[A]` | заменён на `ZIO` / `ZIOAppDefault`; чтение отчёта и аудио — ZIO-эффекты |

### Почему так

- Наивный `Reader` — это `Env => A`. ZIO-окружение `ZIO[R, E, A]` — то же
  самое с типизированными ошибками. `ZIO.service[RenderConfig]` =
  `Reader.ask`, `ZLayer.succeed(cfg)` = подстановка окружения.
- В исходной версии цикл анимации был императивным `while` с `var
  animState` — вынужденная мера, потому что наивный `IO` без trampoline
  переполнял стек на тысячах `flatMap`. Теперь цикл `Animation.loop` —
  рекурсивный ZIO-эффект; ZIO безопасен по стеку, поэтому рекурсия
  loop → loop работает на любой длине трека.
- `zio.Ref` — потокобезопасная изменяемая ячейка; она заменяет и
  `var animState`, и наивный `State`. `Ref.updateAndGet` атомарно
  применяет чистую функцию перехода — ровно то, что делала `State`.
- `AudioPlayer` предоставляет `Clip` через `ZIO.acquireRelease`: при
  выходе из `Scope` клип останавливается и закрывается автоматически,
  даже если визуализация упала с ошибкой.
- `Writer` оставлен в `ReportReader` намеренно — ТЗ просит «в паре мест
  пусть будет».

## Структура проекта

```
src/main/scala/
├── monads/              # Блок 0: собственные наивные монады
│   ├── Monad.scala
│   ├── Reader.scala
│   ├── Writer.scala
│   ├── State.scala
│   └── IO.scala
├── domain/              # Предметная область (на ZIO)
│   ├── Visual.scala         # типы, RenderConfig + ZLayer, AnimationState
│   ├── SimpleJson.scala     # собственный парсер JSON (чистый)
│   ├── ReportReader.scala   # чтение отчёта (ZIO Task + Writer)
│   ├── AudioPlayer.scala    # воспроизведение WAV (ZIO-ресурс)
│   ├── Renderer.scala       # логика кадра (ZIO-окружение + Ref)
│   └── Animation.scala      # цикл анимации (рекурсивный ZIO-эффект)
├── ui/                  # Графический интерфейс ("view")
│   ├── SpectrumCanvas.scala   # анимированная панель отрисовки
│   └── VisualizerWindow.scala # окно, связка Swing ↔ ZIO
└── app/
    └── Main.scala       # ZIOAppDefault: CLI + флаг --ui
```

## Требования

- JDK 17 или 21
- sbt 1.9+
- Scala 3.3.4
- ZIO 2.1.x (подтягивается sbt автоматически)

```bash
# 1. Запустить анализатор (в соседнем проекте):

sbt "run track.wav --json my_report.json"

# 2. Запустить визуализатор:
cd ../visualizer
sbt "run ../analyzer/my_report.json ../analyzer/track.wav"
```
