# Визуализатор аудио

Курсовая по функциональному программированию (Scala 3).

Программа читает JSON-отчёт, созданный анализатором аудио, и визуализирует
спектр в терминале синхронно с воспроизведением исходного WAV-файла.
Поддерживает несколько режимов визуализации, синхронизированный
индикатор бита и переключение режимов на лету.

Работает в паре с проектом `analyzer`, формат отчёта между ними
описан в общем `REPORT_FORMAT.md`.

## Запуск

```bash
# Передать оба пути:
sbt "run ../analyzer/report.json ../analyzer/track.wav"

# С опциями:
sbt "run report.json track.wav --mode bars --width 100 --height 25"

# Интерактивный режим:
sbt run
```

### Опции

| Опция | Значение по умолчанию | Назначение |
|---|---|---|
| `--mode <spectrum\|bars\|spectrogram>` | spectrum | режим визуализации |
| `--width <число>` | 80 | ширина в символах |
| `--height <число>` | 20 | высота в строках |
| `--fps <число>` | 30 | частота кадров |

### Режимы

- **spectrum** — спектр с делением на три цветовые зоны (бас/средние/высокие)
- **bars** — три крупных столбика с пробелами между ними
- **spectrogram** — заглушка для будущей реализации

## Возможности

- Чтение JSON-отчёта от анализатора (свой парсер JSON без библиотек)
- Воспроизведение WAV через `javax.sound.sampled`
- Синхронизация визуализации с реальной позицией воспроизведения
- Цветной ANSI-рендеринг в терминале
- Несколько режимов визуализации (расширяемая система через `VisualMode`)
- Индикатор бита (вспышка `BEAT` в верхней строке)
- Отображение времени, BPM, текущего режима

## Архитектура

Проект построен на собственных монадах без внешних библиотек:

| Монада | Назначение |
|---|---|
| `Monad[M[_]]` | базовая абстракция, операции `pure`, `flatMap`, `map` |
| `Reader[RenderConfig, A]` | проброс настроек рендеринга (размер, режим, fps) |
| `Writer[Vector[String], A]` | накопление логов загрузки отчёта и проверок |
| `State[AnimationState, A]` | состояние анимации: индекс кадра, индикатор бита, режим |
| `IO[A]` | чтение файлов, аудио, рисование в терминале |

### Где какая монада применяется

- **Reader** — `Renderer.renderFrame` зависит от `RenderConfig`,
  `Animation.run` тоже возвращает `Reader[RenderConfig, IO[Unit]]`
- **Writer** — `ReportReader.read` накапливает лог загрузки отчёта
- **State** — `Renderer.tickAnimation` и `Renderer.switchMode` работают
  с `State[AnimationState, ...]`. Это моделирует изменение состояния
  анимации без мутации
- **IO** — `ReportReader.read`, `AudioPlayer.play/stop`, `Renderer.setup/teardown`,
  весь `Main.scala`

## Структура проекта

```
src/main/scala/
├── monads/              # Блок 0: реализация монад
│   ├── Monad.scala
│   ├── Reader.scala
│   ├── Writer.scala
│   ├── State.scala
│   └── IO.scala
├── domain/              # Предметная область
│   ├── Visual.scala         # типы: Report, BandSnapshot, AnimationState, VisualMode
│   ├── ReportReader.scala   # парсер JSON-отчёта (свой парсер)
│   ├── AudioPlayer.scala    # воспроизведение WAV (javax.sound)
│   ├── Renderer.scala       # рендер кадра, State-операции для анимации
│   └── Animation.scala      # цикл рендеринга с синхронизацией
└── app/
    └── Main.scala       # CLI, IO-сценарий
```

## Парсер JSON

В проекте используется **собственный** recursive-descent парсер JSON
(`domain/SimpleJson`). Это сделано намеренно:
- не тянем внешних зависимостей
- демонстрация работы с грамматиками в функциональном стиле
- проще объяснять на защите

Парсер поддерживает все типы JSON-значений: объекты, массивы, строки,
числа, true/false/null, escape-последовательности в строках.

## Требования

- JDK 17 или 21
- sbt 1.9+
- Scala 3.3.4

## Формат отчёта

См. `REPORT_FORMAT.md` в репозитории.

## Пример полного процесса

```bash
# 1. Запустить анализатор (в соседнем проекте):
cd ../analyzer
sbt "run track.wav --json my_report.json"

# 2. Запустить визуализатор:
cd ../visualizer
sbt "run ../analyzer/my_report.json ../analyzer/track.wav"
```
